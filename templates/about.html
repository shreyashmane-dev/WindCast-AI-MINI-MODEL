Mission & Purpose

Optimizing Renewable Energy Integration

Wind power is one of the cleanest sources of electrical energy, but it is highly intermittent and dependent on ambient atmospheric conditions. Sudden drops in windspeed can cause power grid shortages, while high winds require turbines to shut down to prevent mechanical failure. WindCast AI is designed to provide grid operators and wind farm technicians with short-term wind turbine power output forecasts. By utilizing machine learning models to map windspeed, wind direction, temperature, and humidity directly to turbine power output, we enable efficient grid load balancing, reduce dependency on fossil-fuel backup generators, and optimize maintenance schedules.

Technology Stack

Under the Hood

Modern web & machine learning stack

💻 Frontend UI

Built using semantic **HTML5** and custom styled **Vanilla CSS3** to maximize responsiveness and rendering speed. Visual analytics (Power Curve scatters and polar Wind Roses) are drawn natively using **SVG vector mapping**. Dynamic prediction recalculations on model selection are performed asynchronously via the JavaScript **Fetch API (AJAX)**.

⚙️ Flask Backend

The server runs on **Flask** (Python 3.11), providing a lightweight routing mechanism to separate endpoints. Data persistence is managed locally with a lightweight **SQLite3** database which logs prediction parameters and results. Models are serialized using **Joblib** to load weights into memory instantly.

🧠 Machine Learning

Models are trained on a clean dataset of 43,800 records using **Scikit-Learn** (Linear Regression, Decision Tree, Random Forest Regressor), **XGBoost** for gradient-boosted trees, and **TensorFlow/Keras** to implement a sequential **Long Short-Term Memory (LSTM)** neural network for capturing temporal patterns.

Workflow

How WindCast AI Works

01

Load Dataset

The app reads the local wind dataset, keeps the four weather features, removes empty rows, and uses measured power as the target value.

02

Train Models

Classic machine learning models learn from tabular readings. The LSTM path scales the data and builds short rolling sequences for neural time-series learning.

03

Select Best

Every trained model is tested on held-out data. MAE, RMSE, and R2 are calculated, and the lowest RMSE model becomes the default predictor.

04

Track Results

Each submitted forecast is saved in SQLite, shown in the History tab, and converted into trend, bar, and input-strength graphs in the Prediction tab.

Methodology

Data Preprocessing & Sequence Logic

MinMax Scaling

Features are scaled between 0 and 1 using `MinMaxScaler` prior to neural network evaluation. This prevents features with larger numerical ranges (like wind direction) from dominating the learning process.

LSTM Windowing

To capture time dependencies, the dataset is windowed into rolling sequences of length = 6. This allows the LSTM network to analyze the last 30 minutes of weather trend transitions (assuming 5-minute sampling intervals) to forecast the next time step's power output.

Dynamic Model Evaluation

Scores make the app transparent: MAE shows average error, RMSE penalizes large errors, and R2 shows how well the model explains power variation.
